One of the models experimented with uses Hugging Face's XLNetForSequenceClassification architecture initialized with the xlnet-base-cased pre-trained checkpoint and tokenizer, with inputs padded and truncated to a maximum length of 128 tokens to ensure consistent dimensions. During training over 3 epochs with a per-device batch size of 8 (creating an effective batch size of 16 via 2 gradient accumulation steps), the model utilizes the AdamW optimizer, a learning rate of 2e-5, a weight decay of 0.01, and FP16 mixed-precision to accelerate computation. The task's objective function relies on the standard Cross-Entropy Loss inherent to Hugging Face sequence classification models, and the Trainer is configured to automatically load the best-performing weights based on the validation F1-score at the end of the run. The model achieved an 85.21\% accuracy and an 85.21\% macro F1-score, significantly outperforming a simple majority class baseline (F1 = 0.33, Acc = 0.5) but performing very similar to the TF-IDF Logistic Regression baseline (F1 = 0.85, Acc = 0.85). This similarity in performance is notable, while XLNet uses permutation language modeling to capture deep, bidirectional semantic context, TF-IDF relies strictly on word frequencies. The lack of a significant performance gap suggests that the genres in this specific dataset are highly distinguishable through surface-level vocabulary where TF-IDF excels, rather than requiring complex contextual understanding. Alternatively, increasing the sequence length beyond 128 tokens or conducting further hyperparameter tuning might be necessary to have the model improve its performance.
